準備中・追加判断が必要な機能

機能現状打ち合わせで決めたいこと
Lark OAuth ログインルートはあるが実ログインは未接続Lark アプリ情報、許可スコープ、ログイン運用
Lark Base 同期設定 UI は実装済み、同期 API は未接続Base / Table / Field 構成、同期方向、競合時の扱い
Lark Calendar 同期CRM 側カレンダーは実装済み、Lark 同期は未接続Calendar ID、同期対象、担当者別運用
Lark ApprovalCRM 内承認は実装済み、Lark 承認フロー起動は未接続承認フロー ID、承認者、差戻し運用
Lark Bot 通知通知ログ・テンプレートは準備済み、送信 API は未接続通知先チャット、送信タイミング、文面
メール通知準備中メール通知を使うか、Lark に一本化するか
レポート出力グラフ閲覧のみCSV / PDF エクスポートの必要範囲
監査ログ閲覧 UIDB 側の監査ログは想定、閲覧画面は未実装管理者画面で見る必要があるか
パスワード再発行 UIアプリ内ボタンは未実装Supabase 側運用か、管理画面に追加するか
MDRT 目標設定 UI目標値は DB 管理、画面更新は未実装管理者が年度目標を画面で変える必要があるか
タスク管理旧タスクテーブルは撤去済みカレンダー予定で足りるか、独立タスクが必要か
Web会議 URL 専用項目現状はメモ欄で運用専用フィールドを追加するか

打ち合わせでの確認順

  1. 現行画面を見せ、利用できる機能範囲を確認する
  2. Lark 連携をどこまで初回リリースに含めるか決める
  3. スタッフ・商品・保険会社・既存データの提供形式を確認する
  4. 意向把握、承認、高齢者対応、通知の運用ルールを確認する
  5. 追加要望を「初回リリース必須」と「リリース後改善」に分ける
  6. 受入テストの日程、確認担当、完了条件を決める
