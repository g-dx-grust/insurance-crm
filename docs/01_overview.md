# 01. システム概要

## 何のためのシステムか

**N-LIC CRM** は、生命保険・損害保険代理店の業務オペレーションを止めずに、コンプライアンス要件を満たしながら回すための業務 CRM です。
グラスト G-DX 共通ルール（業務 SaaS としての落ち着いた UI / 監査ログ重視 / 運用負荷優先）に従って実装されています。

主なゴール：

1. **顧客と契約の一元管理** — 顧客マスタ・家族構成・契約・特約・更改履歴を一画面でたどれる
2. **保険業法対応の証跡化** — 意向把握記録は承認フロー付きで物理削除禁止
3. **満期・タスク漏れの防止** — 満期 60 日前アラート、ダッシュボード／レポート／通知連携
4. **代理店経営の見える化** — 月次推移・担当者別実績・MDRT 進捗・商品カテゴリ別構成
5. **Lark 連携前提の運用** — SSO、承認フロー、通知 Bot、Base 同期を将来一体運用

## 全体像

### 機能カテゴリ

```
┌─ ホーム ────────────────┐
│  ダッシュボード         │  KPI / 満期アラート
└─────────────────────────┘

┌─ 顧客・契約 ────────────┐
│  顧客管理               │  検索 / 登録 / 家族構成 / 高齢者フラグ
│  契約管理               │  特約 / 更改履歴 / 満期管理
└─────────────────────────┘

┌─ 案件・活動 ────────────┐
│  案件管理               │  カンバン (ステージ別) / 活動履歴
│  カレンダー             │  月・週ビュー
└─────────────────────────┘

┌─ 保険業法対応 ──────────┐
│  意向把握               │  4 ステップウィザード / 承認 / 論理削除のみ
└─────────────────────────┘

┌─ 精算・成績 ────────────┐
│  精算・MDRT 管理         │  CSV インポート / 月次精算 / 目標進捗
└─────────────────────────┘

┌─ 分析・設定 ────────────┐
│  レポート               │  月次・担当者・商品カテゴリ・年代分布
│  Lark 連携              │  SSO / Base / Calendar / Approval / Bot
│  設定                   │  組織 / ユーザー / 通知 / コンプラ
└─────────────────────────┘
```

### ロール (権限) と画面アクセス

| ロール | 主な役割 | 画面アクセス |
|---|---|---|
| `admin` | 代理店オーナー・管理者 | すべて。設定 / ユーザー招待 / Lark 連携 / 監査ログ |
| `agent` | 代理店所属の担当者 | 自分担当の顧客・契約・案件・意向把握。レポート閲覧可 |
| `staff` | 事務スタッフ | 顧客・契約の検索／登録／編集。意向把握の承認は不可 |

> ⚠️ ロール別の表示制御は UI ガード（画面側）と RLS（DB 側）の二重で行われます。仮に URL を直叩きされてもサーバー側で拒否されます。詳細は [14. セキュリティ](./14_security_compliance.md) を参照。

## 技術構成

エンドユーザー向けのドキュメントとして、設定や運用に関わる範囲のみ記載します。

| レイヤー | 採用技術 |
|---|---|
| フレームワーク | Next.js 16 (App Router, Turbopack) |
| 言語 | TypeScript (strict) |
| UI / デザイン | React 19 + Tailwind CSS v4 + shadcn/ui (base-nova) |
| フォント | Noto Sans JP |
| アイコン | Lucide React |
| データベース | Supabase (PostgreSQL) — テナント単位の Row Level Security |
| 認証 | ID + パスワード（Supabase Auth） / Lark OAuth（準備中） |
| 通知 | Lark Bot（準備中） / メール（準備中） |
| デプロイ | Vercel（想定） |

実装上の重要な特徴：

- **論理削除（`deleted_at`）** が基本。顧客・契約・案件は復元可能、意向把握は復元すら原則しません。
- **`tenants.settings`** に JSONB で代理店ごとの設定を保持。Lark 連携設定もすべて DB 管理（ハードコード禁止）。
- **`updated_at` 自動更新トリガー** が主要テーブルに設定済み。

## このシステムが「やらないこと」

導入検討時の判断材料として、明示的に対象外としているスコープを示します。

- 保険会社の見積もり機能・申込書作成は **行いません**（外部の保険会社システム想定）
- 顧客との直接的なメッセージング機能は **行いません**（Lark / メール / 電話を運用で使う想定）
- 会計・税務処理は **行いません**（精算データのエクスポートまで）
