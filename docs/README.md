## 読み方のヒント

- **新しく評価する方** は `01 → 02 → 10`（概要 → 共通操作 → ダッシュボード）の順で読むと全体像が掴めます。
- **現場運用に向けた検証** は `03 → 04 → 06 → 07` の業務フロー順に読むと業務適合性を確認できます。
- **管理者の方** は `11 → 12 → 13 → 14` で運用 / 連携 / 統制を確認できます。

## 表記ルール

- 画面上のクリック対象は **［新規登録］** のように **隅付き括弧** で示します（ボタン・タブ・メニュー）。
- パス・URL・スキーマ名は `等幅` で示します。
- 注意は次の形式で表示します。

  > ⚠️ 物理削除はできません。意向把握記録は保険業法対応のため論理削除（`deleted_at`）のみ。

## 用語

| 用語 | 意味 |
|---|---|
| テナント | 1 つの代理店組織。データ・設定はテナント単位で完全に分離されます (RLS) |
| ロール | `admin` / `agent` / `staff` の 3 種。権限と画面表示が変わります |
| 高齢者 | 設定 > コンプライアンスで定めた年齢以上の顧客。意向把握フローで追加対応が必要 |
| 意向把握 | 保険業法に基づくニーズヒアリング記録。物理削除禁止 |
| MDRT | Million Dollar Round Table。生命保険業界の年間実績指標 |
